**名称**: Grant Proposal Support（助成金キャッチアップBOT）
**目的**: 株式会社HAMAYOUリゾートが使える助成金・補助金を、月1で自動リサーチ→ChatWork通知→役員会議で判断、という流れに乗せる。「知らなかったから使えなかった」をなくす。

**会社プロファイル**:
- 山梨県 富士河口湖町 西湖
- 業種: 宿泊業／観光業／温泉業／キャンプ場運営
- 規模: 中小企業、正社員40〜50名（+パート・技能実習生）
- 主要施設: 光風閣くわるび（団体研修特化ホテル・最大500名）／いずみの湯（日帰り温泉）／キャンプビレッジGnome
- 特殊事情: 旧レイクホテル西湖を他社に賃貸中

**LLM体制**:
- オーナー: Claude Code（設計・コード・正本管理）
- スカウト: **Perplexity Computer**（月次リサーチ担当。今回の相談対象）
- 設計監査役: ChatGPT Pro（あなた）
- 助成金アドバイザー: ChatGPT Pro（会議前に候補をメリデメ分析）

**処理フロー**:
```
月1トリガー（毎月第1金曜 朝7時）
 → Perplexity Computer が助成金リサーチ
 → data/grants/YYYY-MM.md としてGitHubにpush
 → GitHub Actions が ChatWork API に投稿
 → 役員会議の冒頭議題として読み上げ判断
```

**正本管理原則**: 正本は常にGit側。Perplexity Computerは実行環境であり正本ではない。プロンプトは `prompts/` 配下で管理。

## 設計方針の叩き台

**ファイル**: `prompts/monthly-research.md`（固定部分のみ。月別の可変部分は将来必要になったら分離）

**プロンプトの構成**:
1. 役割定義（助成金リサーチャー）
2. 会社プロファイル埋め込み
3. 情報源の巡回指示（sources.md の Tier1 必須、Tier2 はテーマ適合時）
4. 罠チェック通過義務（traps.md を全件通す）
5. スクリーニング基準（明らかに対象外は候補から外す、ただし理由を記録）
6. 出力先（data/grants/YYYY-MM.md にpush）
7. 出力スキーマ（1助成金1ブロックのMarkdown、chatwork-format.md の個別通知項目を全て埋められる粒度）
8. ハルシネーション防止ルール（一次情報URL必須、不明は「不明」と明記、数字を捏造しない）

**出力スキーマに含める項目（案）**:
- 制度名 / 実施主体 / 一次情報URL
- 想定使途（HAMAYOUのどの施設/事業に効くか）
- 締切 / 準備余裕（高中低＋間に合わせる条件）
- 金額感（補助率・上限）
- 事業適合（高中低＋理由）
- 準備余裕（高中低＋理由）
- 仮判断ラベル
- 採る理由 / 実行条件 / 主な負担・制約
- 罠チェック結果（該当した罠と対処方針、該当なしなら「なし」）
- 出典日付（情報を取得した日付）

## 特に評価してほしい観点

1. **項目の過不足**: この出力スキーマで、会議の場で役員が判断するのに足りるか。逆に多すぎて Perplexity が品質を落とす項目はないか。
2. **ハルシネーション耐性**: Perplexity Computer は一次情報URLを必ず取るが、要約時の数値誤認・締切誤認が発生しうる。これを防ぐプロンプト側の仕組みとして、他に何を入れるべきか（例: 「原文引用を付けさせる」「締切は年月日で必ず書かせる」など）。
3. **罠チェックの効かせ方**: traps.md を毎回通させる仕組みとして、「最後にチェックリスト形式で自己検証させる」のが良いか、「各候補に罠該当欄を必ず書かせる」のが良いか、あるいは両方か。実運用で効く書き方は？
4. **仮判断ラベルを Perplexity に付けさせる是非**: 仮判断ラベル（採用候補／条件付き採用／持越し候補／見送り候補）は、リサーチLLMに付けさせるべきか、それとも別レイヤー（後段のフィルタや人間）に任せるべきか。Perplexity に判断を委ねると「採用候補」が乱発される懸念がある。
5. **見落とし**: この設計で取りこぼしそうな構造的リスクはあるか（例: 「該当なし月」に Perplexity が無理に候補を捻出してノイズが増える、など）。

### 最優先の3ガードレール
1. 候補/監視継続/除外 の3区分化（0件月を許容）

- 監査役提案の「前月差分（new/updated/repeated）」は**今回スコープ外**。状態保持の設計が未整備のため、運用が回ってから追加

  - 0件月の完成形マークダウンを先に用意（few-shotとして埋め込み）
