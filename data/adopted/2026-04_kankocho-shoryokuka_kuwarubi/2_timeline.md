# 進捗タイムライン

> **いつ書く？** 状況が変わったとき（申請した、採択された、入金された、など）
> **何を書く？** 下の表で、該当する行に日付とメモを記入してください
> **所要時間:** 30秒。1行書くだけです

---

## 書き方

状況が変わるたびに、該当する行の「日付」と「メモ」を埋めてください。
上から順に進んでいきます。飛ばす行があっても大丈夫です。

| # | ステータス | 日付 | メモ |
|---|-----------|------|------|
| 1 | 会議で採用決定 | 2026-04-11 | （自動記入済み） |
| 2 | 申請準備中 | | 例: DMOに連絡した、見積もり依頼した |
| 3 | 申請済み | | 例: 5/29に提出完了 |
| 4 | 交付決定 | | 例: 7月に通知が届いた |
| 5 | 事業実施中 | | 例: 機器の設置工事開始 |
| 6 | 実績報告済み | | 例: 写真と報告書を提出 |
| 7 | 確定（金額確定） | | 例: 申請額から○万円減額 |
| 8 | 入金 | | 例: ○月○日に振込確認 |
| 9 | 事後報告中 | | 例: 毎年○月に報告義務あり |
| 10 | 義務終了 | | 例: 2031年に全義務完了 |

## よくある補足

- **順番通りに進まないことがある**のは普通です。「交付決定」と「事業実施」が同時期に始まることもあります
- **入金で終わりではありません。** 多くの制度で3〜5年の事後報告義務があります。「義務終了」まで追跡することで、返還リスクを防ぎます
- **メモは短くてOK。** 「提出した」「届いた」程度で十分です。詳しい書類は `evidence/` フォルダに入れてください
